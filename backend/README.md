# Backend

## Overview

This service powers the ChatGPT-style application API and real-time AI chat flow. It provides user authentication, chat creation and sharing, message retrieval, Socket.IO-based AI responses, MongoDB persistence, and Pinecone-backed semantic memory.

## Technologies

- Node.js and Express 5
- MongoDB with Mongoose
- Socket.IO
- JSON Web Tokens (JWT) and `bcryptjs`
- Google Gen AI (`@google/genai`) for Gemini generation and embeddings
- Pinecone for vector-memory storage and retrieval
- CORS, cookies, request logging (`morgan`), and dotenv

## Installation

```bash
cd backend
npm install
```

## Environment Variables

Create a `.env` file in `backend/`. Use your own values; do not commit this file.

```env
PORT=<server_port>
MONGO_URI=<mongodb_connection_string>
FRONTEND_URL=<allowed_frontend_origin>
JWT_SECRET=<long_random_jwt_signing_secret>
GEMINI_API_KEY=<google_genai_api_key>
PINECONE_API_KEY=<pinecone_api_key>
PINECONE_INDEX_NAME=<pinecone_index_name>
```

The Pinecone index must be compatible with the 768-dimensional embeddings generated by `gemini-embedding-2`.

## Run

```bash
npm run dev
```

This starts `server.js` through Nodemon. The server listens on `PORT`, or port `8000` when `PORT` is not set.

## API Endpoints

Protected routes require:

```http
Authorization: Bearer <access_token>
```

### Authentication

| Method | Path | Auth | Description |
| --- | --- | --- | --- |
| `POST` | `/auth/register` | No | Creates an account, returns a 10-minute access token, and sets a 7-day HTTP-only `refreshToken` cookie. Body: `firstName`, `lastName`, `email`, `password`. |
| `POST` | `/auth/login` | No | Validates credentials, returns a 10-minute access token, and sets a 7-day HTTP-only `refreshToken` cookie. Body: `email`, `password`. |
| `POST` | `/auth/logout` | No | Clears the `refreshToken` cookie. |
| `GET` | `/auth/refresh` | Refresh cookie | Validates the `refreshToken` cookie and issues a fresh 10-minute access token plus user details. |

Successful register, login, and refresh responses return `success`, `accessToken`, and `user` (`id`, `fullName`, `email`).

### Chats

| Method | Path | Auth | Description |
| --- | --- | --- | --- |
| `POST` | `/chat` | Yes | Creates a chat for the authenticated user. Body: `prompt`; Gemini generates the chat title. |
| `GET` | `/chat` | Yes | Returns the authenticated user's chats, newest `lastActivity` first. |
| `DELETE` | `/chat/:chatId` | Yes | Deletes an owned chat, its MongoDB messages, and all Pinecone vectors filtered by its `chatId`. |
| `POST` | `/chat/share/:chatId` | Yes | Creates or returns a persistent public sharing URL for an owned chat. |
| `GET` | `/chat/share/:shareToken` | No | Returns the shared chat metadata and its messages when the sharing token is active. |

### Messages

| Method | Path | Auth | Description |
| --- | --- | --- | --- |
| `GET` | `/message/:chatId` | Yes | Returns all messages for the given valid MongoDB chat ID, ordered oldest first, along with `total`. |

### Common Responses

Responses are JSON and generally include `success` and, when relevant, `message`. Implemented error cases include `400` for invalid input, `401` for missing/invalid access tokens, `403` for attempts to delete or share another user's chat, `404` for missing resources, and `500` for server failures.

## Database Models

### User (`User`)

| Field | Type / constraints |
| --- | --- |
| `fullName.firstName` | String, required, maximum 20 characters |
| `fullName.lastName` | String, required, maximum 20 characters |
| `email` | Unique, required string |
| `password` | Required string, minimum 6 characters; stored as a bcrypt hash |
| `createdAt`, `updatedAt` | Automatic timestamps |

### Chat (`chat`)

| Field | Type / constraints |
| --- | --- |
| `user` | Required reference to the owning user |
| `title` | Required string, generated from the first prompt |
| `shared` | Boolean, defaults to `false` |
| `sharedLink` | String or `null`; stores the share token |
| `lastActivity` | Date, defaults to creation time and is updated for AI messages |
| `createdAt`, `updatedAt` | Automatic timestamps |

### Message (`message`)

| Field | Type / constraints |
| --- | --- |
| `chat` | Required reference to a chat |
| `user` | Required reference to the sending user |
| `content` | Required message text |
| `role` | `user`, `model`, or `system` |
| `promptTokens`, `completionTokens`, `totalTokens` | Numbers, default `0`; recorded after an AI response |
| `createdAt`, `updatedAt` | Automatic timestamps |

## Authentication

Passwords are hashed with bcrypt using 10 salt rounds. Access tokens carry `{ userId }`, are signed with `JWT_SECRET`, and expire after 10 minutes. A separate refresh token expires after 7 days and is stored in the HTTP-only `refreshToken` cookie.

REST middleware validates the `Bearer` access token, loads the user from MongoDB, and exposes it as `req.user`. Socket.IO accepts the access token through `socket.handshake.auth.token`; it also has a development fallback for an `accessToken` cookie.

## Socket.IO

Socket.IO runs on the same HTTP server and permits `FRONTEND_URL` with credentials. The handshake must include a valid access token:

```js
io(BACKEND_URL, {
  auth: { token: accessToken },
  withCredentials: true,
});
```

### Client-to-server events

| Event | Payload | Description |
| --- | --- | --- |
| `ai-message` | `{ chat, content, thinkingLevel, stream }` | Saves the user message, retrieves memory, generates an AI answer, persists both messages, and stores their embeddings. `thinkingLevel` must be `Minimal`, `Low`, `Medium`, or `High`; `stream` controls streaming. |

### Server-to-client events

| Event | Payload | Description |
| --- | --- | --- |
| `ai-chunk` | `{ chat, content }` | Emitted repeatedly for streamed response text. |
| `ai-response` | `{ chat, content, promptTokens, completionTokens, totalTokens, totalResponseTime }` | Emitted when generation completes. |
| `ai-response-error` | `{ error }` | Emitted for invalid payloads, missing chats, or generation failures. |

## AI Integration

Gemini model `gemini-3.1-flash-lite` generates chat titles and responses. The application passes a configured system instruction and maps the requested thinking level to the Google Gen AI SDK's `ThinkingLevel` setting.

For every user and AI message, `gemini-embedding-2` creates a 768-dimensional vector. The vector is stored in Pinecone using the MongoDB message ID. On an AI turn, the backend combines:

1. Up to 20 chronological messages from the current chat (short-term memory).
2. Up to 3 semantically similar vectors filtered to the current user (long-term memory).
3. Context-priority instructions, with current-chat history prioritized over vector memory.

## Security

The current implementation includes bcrypt password hashing, JWT verification, HTTP-only refresh cookies, CORS restricted to `FRONTEND_URL`, and ownership checks before a chat can be deleted or shared.

Before production deployment, address these important hardening items:

- Set cookie options such as `secure: true`, `sameSite`, and an explicit `path` appropriate for the deployment; the current refresh cookie only sets `httpOnly` and `maxAge`.
- Validate and normalize all request and socket payloads with a schema validator. The current handlers perform only limited required-field checks.
- Verify chat ownership in `GET /api/message/:chatId` and in the `ai-message` socket handler. Currently, an authenticated user can request messages for, or submit a message to, a chat ID they do not own.
- Confirm a socket user was found after JWT verification; the current handshake checks the token but does not reject a deleted user explicitly.
- Add rate limiting, secure headers (for example Helmet), centralized error handling, request-size limits, observability, and dependency/security monitoring.
- Use TLS in production, keep all API keys in a secret manager, rotate `JWT_SECRET` and provider keys, and never log credentials or tokens.
- Treat public share links as bearer secrets. There is no revocation endpoint; a shared chat remains publicly readable while its token is valid and `shared` is true.
